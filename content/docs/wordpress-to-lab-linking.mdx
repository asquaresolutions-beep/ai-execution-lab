This document maps the link flow from `asquaresolution.com` WordPress posts to specific content on `lab.asquaresolution.com`. Links are categorized by post topic type, anchor text pattern, and priority tier.

Execute high-priority links first — they pass the most authority and are the most topically relevant. Work through medium and low priority in subsequent passes.

---

## The Linking Principle

Every link from WordPress to the Lab should be:

1. **Contextually earned** — the linked content meaningfully extends what the WordPress post says
2. **Editorially natural** — the anchor text reads as part of the sentence, not an SEO insertion
3. **Non-reciprocal as the primary motive** — link because the content adds value to the reader, not because you want a backlink

Maximum 2 outbound Lab links per WordPress post. If a post has multiple relevant Lab references, choose the two most relevant.

## Contextual Anchor Text — Pattern Guide

### What Makes a Good Anchor

Good anchor text:
- Reads naturally in the sentence
- Describes what the linked page actually covers
- Is specific (not "click here", "read more", "this article")
- Varies between uses (don't use the same phrase for every Lab link)

Bad anchor text:
- "click here" / "learn more" / "read more"
- The full URL as anchor text
- Keyword-stuffed phrases that don't read naturally
- Identical anchor text across multiple links to the same page

### Phrase Patterns

**Linking to a track:**
> "...documented in the [Track Title] track at AI Execution Lab."
> "...which we cover in full in the [Track Title] track."
> "...the full execution guide is at lab.asquaresolution.com → [Track Title]."

**Linking to a specific lesson:**
> "...the exact workflow for this is published here."
> "...we've documented this pattern in detail."
> "...see the [Lesson Title] lesson for the implementation."

**Linking to a failure report:**
> "...we hit this exact failure — root cause and fix documented here."
> "...if you see this error, the cause is documented in the failure archive."

**Linking to a playbook:**
> "...the full playbook for this is at AI Execution Lab."
> "...we published the step-by-step system for this."

## Implementation Method: WordPress

### Option A: Manual edit in WordPress editor (Gutenberg)

1. Open the post
2. Find the relevant paragraph
3. Select the anchor text phrase
4. Add link to the Lab URL
5. Check "Open in new tab" for cross-domain links
6. Update post

### Option B: Bulk linking via WP REST API (for multiple posts)

If you have 20+ posts to update, use the content patching system documented at:
`/playbooks/wp-rest-api-automation-playbook`

Pattern: Read → identify target sentence → insert anchor tag → PATCH via REST API → verify.

---

## Link Audit: Maintenance Schedule

**Monthly:** Review Lab content additions (new lessons, docs, playbooks) against WordPress post categories. Add new links where relevant.

**Quarterly:** Check that all existing Lab links still resolve (no 404s after content restructuring).

**On new Lab publication:** Before publishing any new Lab piece, check if 2–3 existing WordPress posts should link to it. Add those links at publish time.

---

## What Not to Link From WordPress

Avoid linking to:
- `/ops` (internal ops dashboard — noindex, internal tool)
- `/syndicate` (internal tool)
- `coming-soon` lessons (they render a coming-soon state — no value to readers)
- Any Lab page that is draft or thin

Before linking to a specific Lab page, verify it loads and has complete content.
