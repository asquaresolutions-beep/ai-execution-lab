This document specifies the architecture for 12 new execution tracks. All tracks follow the execution-first, implementation-before-theory framework established by existing tracks. Each track is designed as a standalone entry point for its target audience, while cross-linking to the broader ecosystem.

Implementation priority order is specified for each track. Tracks are added to `lib/tracks.ts` as `status: 'available'` as modules are built.

## Implementation Notes

### Adding tracks to lib/tracks.ts

Tracks are added to `lib/tracks.ts` as full entries when at least one module has at least 2 lessons written and published. Do not add a track with 0 available content — it creates a poor first impression and wastes crawl budget.

**First batch for lib/tracks.ts addition (Q3 2026):**
1. `ai-for-non-developers` — when Module 1 is complete (4 lessons)
2. `solo-ai-founder` — when Module 1 is complete (4 lessons)
3. `ai-freelancing` — when Module 1 is complete (4 lessons)

**Second batch (Q4 2026):**
4. `ai-for-students` — when Module 1 is complete
5. `ai-research-operations` — when Module 1 is complete

### Lesson content standards for new tracks

All lessons in new tracks must follow the same standards as existing content:
- `LessonMeta` with difficulty, implementationTime, and evidence fields
- `LessonObjectives` with 4-6 specific, verifiable objectives
- Implementation sections (not conceptual explanations)
- `Checkpoint` at end with verifiable completion criteria
- Cross-links to related failures, playbooks, and lessons where relevant
- Global pricing in USD (not India-specific)
- No country-specific examples unless explicitly scoped
