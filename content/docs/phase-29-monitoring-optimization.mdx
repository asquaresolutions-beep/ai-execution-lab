Post-launch operating workbook. Live metrics (indexing counts, Discover impressions, RPM) live in **your** GSC / AdSense / Vercel / Analytics — this doc tells you exactly what to read, where, the healthy range, and what to do. Sections seeded with priorities **computed deterministically** from the engine (no live data needed). **No new systems — monitoring + high-ROI optimization only.**

> Note: the engine's `/api/health` reports live AI/persistence mode. In production it should read `ai: live`, `persistence: firestore`, `env.ok: true`. (A local sandbox reads mock/memory — that is expected and not the production state.)

## A) Indexing monitoring — GSC → Pages / Sitemaps
| Track | Healthy | Action if not |
|---|---|---|
| Indexed page growth | rising toward ~250 over 2–3 wks | request-index stalled tier-1 |
| Excluded / "Discovered – not indexed" | shrinking | improve internal links / content depth |
| Sitemap coverage | submitted, ~730 discovered | resubmit; check origin matches canonical |
| Crawl anomalies (5xx/redirects) | none | check Vercel logs / canonical config |
| Orphan risk | 0 | re-run crawl-depth after big content adds |

## B) Discover monitoring — GSC → Performance → Discover
| Track | Healthy | Action |
|---|---|---|
| Discover impressions | first by wk 2–3, then rising | push freshest tier-1 + in-season hub |
| Discover CTR | improving | strengthen headline (honest) + OG image |
| Strongest Discover pages | identify top 5 | make more like them; interlink |
| Freshness decay | refresh pages older than ~30d | update "last updated" + trending strip |
| Image/OG effectiveness | large-image previews showing | confirm robots allows /api/og |

## C) SEO/GEO monitoring — GSC Performance + manual spot-checks
- **Featured snippets / AI Overview:** spot-check tier-1 "what is / is this a scam" queries weekly; the direct-answer + comparison + speakable schema target these.
- **FAQ visibility:** GSC → Enhancements (FAQ) — fix any errors.
- **Entity growth + semantic expansion:** `/api/scam-intel/growth?view=entities` (fastest-growing categories/platforms/regions).
- **Internal-link effectiveness:** pages with rising impressions that you linked from the hub — keep depth ≤3.

## D) Content optimization
- **Strongest entities:** highest GSC clicks/impressions → expand with city combos + Hindi.
- **Weakest pages:** high impressions + low CTR → rewrite title/headline (truthful) first; thin/low-severity → consider deprioritising.
- **CTR-hook candidates:** pages with impressions but CTR below site median.
- **Freshness refreshes:** tier-1 pages older than 30d; bump dates + re-promote.
- **Internal-link opportunities:** link new trending pages from the hub + related type pages (0 orphans, depth ≤3).

## E) Distribution optimization
| Track | Where | Use |

## F) Monetization monitoring — AdSense + analytics (only after ads enabled)
- **RPM / CTR / placement:** below-verdict vs in-content unit performance; keep the higher-RPM placement, drop dead weight.
- **Free→paid:** watch which scam topics precede upgrades (Investment/UPI/KYC score highest on intent); position Pro on those pages.
- **Guardrail:** no intrusive/anchor formats; confirm CLS stays ≈0 with ads live.

## G) Cost monitoring — /ops/analytics + GCP billing
| Track | Healthy | Action |
|---|---|---|
| Vertex spend today | under `DAILY_BUDGET_USD` | breaker auto-halts; lower caps if near |
| Quota utilization | under 85% | back off autopilot frequency |
| Firestore reads/writes | low (counters, cached snapshot) | investigate spikes; avoid scans |
| Cache hit rate | rising | ensure cache TTL not bypassed |
| Queue efficiency / DLQ | failure under 5%, DLQ 0 | inspect + replay from DLQ |
| Unnecessary AI usage | none (dedup + quality gate) | confirm crons early-exit when idle |

## H) Weekly optimization loop (highest-ROI)
Run weekly; most inputs are one API call or one GSC view.
1. **Content priorities:** `/api/scam-intel/growth?view=leverage` → publish/promote top movers (baseline: Investment, UPI, KYC, Phishing, Fake-job).
2. **Discover opportunities:** freshest tier-1 + in-season hub (festival Oct–Nov/Mar, tax Mar–Apr, election around polls).
3. **Backlink opportunities:** `?view=backlink` (baseline: UPI, OTP, KYC) → pitch the widget to the matching vertical.
4. **Hindi opportunities:** the tier-1 Hindi pages (already built) with rising EN demand → promote Hindi share copy.
5. **AI Overview opportunities:** tier-1 question queries showing impressions but no snippet → tighten the direct-answer block (no new system).

## I) Guardrails (do NOT add unless metrics justify)
- **No new infra / expensive scaling / redesigns / realtime systems** unless a measured bottleneck appears:
  - Cost spike → lower caps (`DAILY_BUDGET_USD`, `AUTOPILOT_PER_RUN`), not bigger infra.
  - Crawl stall → add internal links, not a new crawler.
  - Throughput limit → split AI calls to a worker only if the queue genuinely backs up (DLQ growing).
  - Vector search slow only above a few thousand docs → switch to Firestore Vector Search (indexes already defined).
- **Default posture:** publish consistently, distribute the deterministic copy, monitor weekly, let the four loops compound (detect→publish→reports; freshness→Discover; widgets→backlinks→authority; trust→AI-citations→brand).
